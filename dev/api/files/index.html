ConstrainedCobbDouglas.load_ohlc_jld2 — Function
load_ohlc_jld2(path) -> NamedTuple

Load a lectures-style OHLC JLD2 file. Returns (prices, dates, tickers, volumes) where prices and volumes are T × K matrices, dates is a Vector{Date} of length T, and tickers is a Vector{String} of length K.

Schema actually used by the lectures repo: the file has a single top-level key "dataset" whose value is a Dict{String, DataFrame} keyed by ticker. Each per-ticker DataFrame has columns open, close, high, low, volume, timestamp, volume_weighted_average_price, number_of_transactions. The timestamp column is DateTime at 05:00:00 UTC; we project to Date.

Only tickers with the modal (maximum) number of rows and timestamps matching the reference timeline are kept — i.e. tickers with partial coverage (IPO/delisting) are dropped here so that the returned matrices are rectangular. Downstream code can apply additional universe filters.

Falls back to the older flat-key schema ("close", "dates", etc.) when the "dataset" key is absent, so legacy fixtures still load.
