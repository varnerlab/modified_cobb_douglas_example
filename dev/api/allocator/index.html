Strategy types

ConstrainedCobbDouglas.MyAllocationStrategy — Type
MyAllocationStrategy

Abstract supertype for all allocation strategies in the backtest harness. Subtypes S <: MyAllocationStrategy must implement two dispatch methods: allocate(::S, state, t, env) returning target positions/weights, and should_decide(::S, state, t) returning a Bool that gates when the allocator runs on day t.

source
ConstrainedCobbDouglas.EqualWeightStrategy — Type
EqualWeightStrategy

Strategy 1 (spec §6.2): passive equal-weight buy-and-hold on the assembled basket. Allocates once at t = 1 with wᵢ = 1/N and holds for the entire horizon (no rebalance, no MPC).

source
ConstrainedCobbDouglas.MinVarBuyHoldStrategy — Type
MinVarBuyHoldStrategy

Strategy 2 (spec §6.2): minimum-variance portfolio computed at t = 1 from the SIM-implied covariance Σ, then held without rebalance. Long-only, fully-invested baseline that isolates the value of variance-minimization without dynamics.

source
ConstrainedCobbDouglas.UnconstrainedCDStrategy — Type
UnconstrainedCDStrategy

Strategy 3 (spec §6.2): analytical Cobb-Douglas allocator (closed-form wᵢ ∝ γᵢ) rebalanced daily. Replicates the live engine's allocator at a daily cadence and serves as the unconstrained, no-trigger benchmark for the constrained / MPC-gated variants.

source
ConstrainedCobbDouglas.CostAwareMVStrategy — Type
CostAwareMVStrategy

Strategy 4 (spec §6.2): mean-variance allocator with a γ-tilt in the expected-return term and an l1 turnover penalty in the objective. Field κ is the risk-aversion coefficient on the quadratic variance term; c is the per-unit l1 turnover cost.

source
ConstrainedCobbDouglas.CDWithMPCStrategy — Type
CDWithMPCStrategy

Strategy 5 (spec §6.2): unconstrained Cobb-Douglas allocator gated by the MPC trigger. Re-allocates only when the MPC band fires (band exit, horizon elapsed, or drawdown circuit-breaker); otherwise holds. Field spec is the MyMPCSpec trigger configuration.

source
ConstrainedCobbDouglas.ConstrainedCDWithMPCStrategy — Type
ConstrainedCDWithMPCStrategy

Strategy 6 (spec §6.2): the new design under evaluation. Cobb-Douglas allocator subject to a covariance cap (σ_max), l1 turnover budget (K_turnover), and per-name concentration cap (w_max), gated by the MPC trigger (spec). Combines the variance/turnover/concentration constraints of Strategy 4 with the event-driven cadence of Strategy 5.

source

Solvers

ConstrainedCobbDouglas.solve_constrained_cd — Function
solve_constrained_cd(problem::MyConstrainedCDProblem) -> MyConstrainedCDResult

Solve max Σ γᵢ log(nᵢ) s.t. budget, covariance, turnover, and concentration constraints (spec §4). Uses Clarabel via JuMP; falls back to SCS if Clarabel returns a non-OPTIMAL status.

Non-preferred assets (γᵢ ≤ 0) are pinned at ε = 1e-3 shares; the optimizer runs over the preferred subset only (keeps the objective concave).

source
ConstrainedCobbDouglas.solve_unconstrained_cd_analytical — Function
solve_unconstrained_cd_analytical(γ, p, B; ε = 1e-3) -> (shares, cash)

Closed-form Cobb-Douglas allocator. Non-preferred assets (γᵢ ≤ 0) get ε shares; preferred assets get proportional allocation.
